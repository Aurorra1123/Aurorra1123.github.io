模板：

做题思路模板

1. 确定dp数组（或变量）以及下标的含义
  • 分析问题，明确dp数组或变量所代表的含义。
  • 对于一维动态规划，dp[i]通常代表到达第i个状态的最优解。
  • 对于二维动态规划，dp[i][j]可能代表到达某个状态（由i和j两个参数定义）的最优解。
2. 确定递归公式（状态转移方程）
  • 根据问题类型，确定当前状态（dp[i]或dp[i][j]）与前一状态或多个状态之间的关系。
  • 递归公式是动态规划解题过程中的核心，它决定了如何从已知的解推导出未知的解。
3. dp数组如何初始化
  • 根据问题和dp数组的定义，确定初始状态的值。
  • 在一些问题中，可能需要将dp数组初始化为特定的值以正确反映问题的初始状态。
4. 确定遍历顺序
  • 确定dp数组填充的顺序，这通常取决于问题的特性和递归公式。
  • 遍历顺序可能是从前向后、从后向前、斜着遍历等。
5. 举例推导dp数组
  • 通过具体的例子验证dp数组的递推过程是否正确。
  • 推导可以帮助理解递归公式的正确性及其对问题的解决。

背包问题

对于背包问题的整体把握：

其中，只需要重点把握一下01背包、完全背包和多重背包。

01背包问题基础（二维数组dp）

有n件物品和一个最多能背重量为w 的背包。第i件物品的重量是weight[i]，得到的价值是value[i] 。每件物品只能用一次，求解将哪些物品装入背包里物品价值总和最大。

此处每一件的物品只有两个状态，取或者不取。

接下来使用动规五部曲来分析这个问题：

1.确定 dp 数组以及下标含义

dp[ i ] [ j ] 表示从下标为[ 0 - i ] 的物品里任意取，放进容量为 j 的背包，价值总和最大是多少。

2. 确定递推公式（初始化与递推公式紧密的联系在一起）

在此处，想要确定 dp [i] [j]的含义，则有两个方向来推出：

  • 不放物品i：由 dp [i-1][j]推出，即背包容量为j，里面不放物品i的最大价值，此时dp[i][j]就是dp[i - 1][j]。(其实就是当物品i的重量大于背包j的重量时，物品i无法放进背包中，所以背包内的价值依然和前面相同。)
  • 放物品i：由dp[i-1] [j-weight[i]] 推出，dp[i-1] [j-weight[i]] 为背包容量为j - weight[i]的时候不放物品i的最大价值，那么 dp [ i - 1] [ j - weight [i]] + value [i] （物品i的价值），就是背包放物品i得到的最大价值

所以递归公式：
$$
dp [ i ] [ j ] = max( dp [ i - 1] [ j ], dp [ i - 1] [ j - weight[ i ]] + value[ i ])；
$$
观察可以发现，仅由上方和左上方的数值来推出。

3. dp数组初始化

因此，需要初始化第一行和第一列的内容。其余地方仅仅为了统一而一致初始化为0.

4.遍历的顺序

在本题中，通过手动演示（这一点非常重要，在一些非常复杂的问题中）你可以发现，无论是先遍历物品还是先遍历背包对结果没有任何影响，因为只需要左上角区域部分的方块就可以得到结果（不一定就是狭义上的左上角）

做动态规划的题目，最好的过程就是自己在纸上举一个例子把对应的dp数组的数值推导一下，然后在动手写代码！——代码随想录

5.代码实现：

01背包问题基础（滚动一维数组）

在此处需要理解为什么能够将二维数组压缩成为一维数组。而这个过程正是实现了一个空间优化的过程

先看二维数组的递推公式：
$$
dp [ i ] [ j ] = max( dp [ i - 1] [ j ], dp [ i - 1] [ j - weight[ i ]] + value[ i ])；
$$
会发现其实得到 DP[i][j]是由二维数组中的上一行数据得到的，因此滚动数组的由来，需要满足的条件是上一层可以重复利用，直接拷贝到当前层。

1.确定dp数组的定义

在一维dp数组中，dp[j]表示：容量为j的背包，所背的物品价值可以最大为dp[j]。

2.一维dp数组的递推公式

大体上与二维数组的公式一致

公式如下：
$$
dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);
$$

3.初始化

此处的初始化难度不高，正常初始化就可以了。

dp[0]=0含义为，容量为0的书包，所背物品的价值最大可以为0。

4.遍历顺序

遍历的时候要求先遍历物品再遍历背包。

在遍历背包的时候，背包是从大到小的倒序遍历，因为能够保证物品i只被放入一次。

举一个例子：物品0的重量weight[0] = 1，价值value[0] = 15

如果正序遍历

dp[1] = dp[1 - weight[0]] + value[0] = 15

dp[2] = dp[2 - weight[0]] + value[0] = 30

此时dp[2]就已经是30了，意味着物品0，被放入了两次，所以不能正序遍历。

为什么倒序遍历，就可以保证物品只放入一次呢？

倒序就是先算dp[2]

dp[2] = dp[2 - weight[0]] + value[0] = 15 （dp数组已经都初始化为0）

dp[1] = dp[1 - weight[0]] + value[0] = 15

所以从后往前循环，每次取得状态不会和之前取得状态重合，这样每种物品就只取一次了。
